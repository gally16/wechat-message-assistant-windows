## 🔧 修复的问题

### 1. AttributeError: 'NoneType' object has no attribute 'isatty'

**问题描述**：打包后的 EXE 在新电脑上运行时，`monitor_web.py` 在模块级别加载配置时尝试访问 `sys.stdin.isatty()`，但在 PyInstaller 打包环境中 `sys.stdin` 为 `None`。

**解决方案**：
- 创建了独立的解密核心模块 `wx_decrypt_core.py`
- 该模块零模块级别副作用，只有常量和函数定义
- 从 `monitor_web.py` 中提取了核心解密函数：
  - `decrypt_page()` - 解密单个加密页面
  - `full_decrypt()` - 全量解密数据库
  - `decrypt_wal_full()` - WAL 增量解密

**修改的文件**：
- 新建：`wx_decrypt_core.py`
- 修改：`wx_gui_notifier.py`（移除动态导入，改为导入 `wx_decrypt_core`）

### 2. Inno Setup 编译错误：Unknown identifier 'DownloadFile'

**问题描述**：`DownloadFile` 函数需要特定的 Inno Setup 版本或插件支持。

**解决方案**：
- 移除了自动下载 VC++ 运行库的复杂逻辑
- 简化了安装脚本，专注于核心安装功能
- VC++ 运行库改为由用户手动安装（如有需要）

### 3. Inno Setup 编译错误：Unknown identifier 'IsWindowsVersionOrNewer'

**问题描述**：`IsWindowsVersionOrNewer` 不是 Inno Setup 6.x 的标准内置函数。

**解决方案**：
- 移除了自定义的 Windows 版本检查代码
- 使用 `[Setup]` 部分的 `MinVersion` 指令进行系统要求检查
- 简化了 `InitializeSetup()` 函数

## 📦 打包步骤

### 方法 1：使用打包脚本（推荐）

```bash
python build_msi.py
```

该脚本会自动：
1. 检查前提条件（PyInstaller、Inno Setup）
2. 使用 PyInstaller 打包 EXE
3. 使用 Inno Setup 创建安装包
4. 显示打包总结

### 方法 2：手动打包

#### 2.1 打包 EXE

```bash
python -m PyInstaller WxGuiNotifier.spec --noconfirm --clean
```

#### 2.2 创建安装包

```bash
& "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" "WxGuiNotifier.iss" /O"installer_output"
```

## ⚠️ 注意事项

### 系统要求
- Windows 10 或更高版本
- .NET Framework 4.7.2 或更高版本（程序会检查，但不强制）
- VC++ 2015-2022 Redistributable（如未安装需手动安装）
